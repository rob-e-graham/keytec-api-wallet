MVP Runtime Path

Store once. Attach to project. Inject at runtime.

The preferred command is intentionally short enough to beat a handwritten .env file in everyday work.

$ famtec add together
OK token saved to keychain
$ famtec run openclaw -- npm run dev
OK API keys injected
values masked, no file written

Activity Ledger

1. Store

Provider handles

Add keys once using provider names. The CLI normalises handles into environment variables.

2. Attach

Project profiles

Profiles become the durable replacement for project-level .env files.

3. Run

Ephemeral injection

The child process receives credentials in memory, then the reference is discarded.

Token Handles

This module shows which provider handles are attached to local profiles. Real credential values never render in the UI, and the dashboard does not enumerate Keychain items.

Project Profiles

Profiles group token handles into runtime-ready bundles. One project profile can replace one local .env file.
